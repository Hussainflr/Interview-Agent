# LobeChat Frontend Integration

The project now vendors the LobeChat source code directly in `frontend/`. The previous custom Next.js frontend has been removed.

## Strategy

LobeChat is the main product frontend. The Python Interview Agent backend exposes an OpenAI-compatible endpoint:

```text
GET  /v1/models
POST /v1/chat/completions
```

The integrated LobeChat source is configured at runtime to use the backend as an OpenAI-compatible provider:

```text
OPENAI_PROXY_URL=http://127.0.0.1:8000/v1
OPENAI_MODEL_LIST=-all,+interview-agent-local=Interview Coach,+interview-agent-hybrid=Interview Coach Hybrid
KEY_VAULTS_SECRET=<generated by run.sh/setup.sh>
AUTH_SECRET=<generated by run.sh/setup.sh>
DATABASE_DRIVER=node
DATABASE_URL=postgresql://postgres:<password>@127.0.0.1:54322/lobechat
```

This keeps the frontend enterprise-grade while keeping the interview workflows, local-first routing, MCP controls, and scoring logic in Python.

## Run Source Frontend

```bash
./run.sh
```

Open:

```text
http://localhost:3010
```

`run.sh` starts:

- FastAPI backend on `127.0.0.1:8000`
- integrated LobeChat source frontend on `localhost:3010`

On first run it also creates stable local `KEY_VAULTS_SECRET` and `AUTH_SECRET` values in `.env`. LobeChat needs these to encrypt stored provider keys and auth/session data.

LobeChat source mode also needs Postgres with `vector` and `pg_search` extensions. If `DATABASE_URL` is not already set, `run.sh` starts a local ParadeDB Postgres service named `lobechat-postgres`, waits for it, runs LobeChat migrations, and then starts the frontend.

## Run Docker

```bash
docker compose up --build
```

Open:

```text
http://localhost:3210
```

Docker builds the local `frontend/` source instead of pulling a generic LobeChat image.

## Local Model Flow

By default the backend uses:

```bash
LOCAL_LLM_PROVIDER=ollama
LOCAL_LLM_MODEL=qwen3:4b
PRIVACY_MODE=local_only
```

Start Ollama:

```bash
ollama pull qwen3:4b
ollama serve
```

In LobeChat, select `Interview Coach`. Messages are sent to the backend, where the agent:

1. Routes irrelevant/greeting input away from scoring.
2. Scores interview answers using local-first model routing.
3. Returns rubric feedback, a stronger answer, and a follow-up question.

## Recommended LobeChat System Prompt

Create an assistant in LobeChat with this system prompt:

```text
You are an AI Interview Coach. Run realistic mock interviews. Ask one question at a time. When the candidate answers, provide scoring, rubric feedback, a stronger answer, and one adaptive follow-up question. Support technical, behavioral, HR, leadership, system design, ML/AI, and UN/INGO interviews. Keep private resume and job-description details local whenever possible.
```

## Where To Customize

- Backend interview behavior: `agents/`, `prompts/`, `providers/`, `mcp/`
- LobeChat frontend source: `frontend/`
- Interview frontend notes: `frontend/INTERVIEW_AGENT.md`
- Provider settings: `.env`, `configs/settings.yaml`

Keep interview-specific intelligence in the backend unless the change is genuinely UI-specific.
